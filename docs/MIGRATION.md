# Migration Guide — from Marzban / 3X-UI to KIAN V2Ray

KIAN can import existing users so you don't start from scratch.

## From Marzban

1. Export your users (Marzban admin → export, or dump the `users` table to JSON):
   ```json
   {"users": [{"username": "ali", "data_limit": 107374182400,
               "expire": 1735689600, "status": "active"}]}
   ```
2. Import:
   ```bash
   python3 scripts/migrate-import.py --marzban marzban-export.json \
     --db /etc/kian-v2ray/kian.db
   ```

## From 3X-UI / x-ui

Point the importer at the x-ui SQLite database:

```bash
python3 scripts/migrate-import.py --xui /etc/x-ui/x-ui.db \
  --db /etc/kian-v2ray/kian.db
```

The importer reads `client_traffics` (email → name, total → quota,
`expiry_time` ms → expiry, `enable` → enabled).

## What is mapped

| Source field | KIAN field |
|--------------|-----------|
| username / email | `name` |
| data_limit / total | `quota_bytes` |
| expire / expiry_time | `expires_at` |
| status / enable | `enabled` |

Existing KIAN users with the same name are **skipped** (never overwritten).
After import, run `kian-v2ray status` (or the panel) to verify, then regenerate
each user's config/subscription.

> Passwords/UUIDs are regenerated by KIAN (keys are client-side by design), so
> users get fresh links — distribute the new subscription links after migration.
